Screenshots

A screenshot always belongs to an application.

Example

http://ws-api.cloudaccess.net/ws/www/screenshots/:screenshot_id

Fields

The Screenshot object has the following fields.

Name Description Returns
id The screenshot's ID integer
image_url URL to the screenshot image string

GET

GET requests can be made to /screenshots in order to get all screenshots or to /screenshots/:screenshot_id in order to get information about a specific screenshot of an application.

Parameter Description Default
application_id The application's ID to which the screenshots belong -

If the request is successful, you get the following return.

Description Type Response Code
If request succeeded to /screenshots Structure containing count field and an array of objects with id and image_url fields 200
If request succeeded to /screenshots/:screenshot_id Structure containing id and image_url fields 200

POST

POST requests can be made to /screenshots to add new screenshots or to /screenshots/:screenshot_id to make DELETE requests if _method parameter exists in query.

Parameter Description Type Default Required
application_id The application's id integer - yes
screenshots[] An array of screenshots array of multipart/form-data - yes

If the create is successful, you get the following return.

Description Type Response Code
If the create succeded boolean 200

PUT

PUT requests are not available for screenshots.

DELETE

DELETE requests can be made to /screenshots/:screenshot_id to delete a screenshot or to /screenshots to delete all screenshots. If your client does not support the DELETE method you can use POST requests with the parameter _method=DELETE.

Parameter Description Type
application_id The application's id integer
ids A list of screenshot_ids separated by comma string

If the delete is successful, you get the following return.

Description Type Response Code
If delete succeded boolean 200
