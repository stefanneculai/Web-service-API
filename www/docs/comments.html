Comments

A comment always belongs to an application and a user.

Example

http://ws-api.cloudaccess.net/ws/www/comments/:comment_id

Fields

The Comment object has the following fields:

Name Description Returns
id The comment'ID integer
message The comment's text string
user_id The comment's author integer
created_at The created date A structure with date, timezone_type and timezone fields
modified_at The date when the comment was modified last time A structure with date, timezone_type and timezone fields
hits The number of comment's hits integer
likes The comment's likes Structure containing a data object and the count of total likes, with data containing an array of objects, each with the state and user_id of the like.

GET

GET requests can be made to /comments in order to get all comments or to /comments/:comment_id in order to get information about a specific comment.

Parameter Description Default
fields A string with the returned fields separated by comma. We recommend to request only the necessary fields in order to save computing time. All
since A string in date format (yyyy-MM-dd) representing the minimum created date 01-01-1970
before A string in date format (yyyy-MM-dd) representing the maximum created date NOW
offset An integer representing the first result 0
limit An integer representing the number of results
Note: The maximum results that can be returned are 100. 		20
application_id The appplication's ID of the returned comments -
user_id The user's ID of the returned comments -

If the request is successful, you get the following return.

Description Type Response Code
If request succeeded to /comments array of objects containing the requested fields 200
If request succeeded to /comments/:comment_id Structure containing the requested fields 200

POST

POST requests can be made to /comments in order to create a new comment or to /comments/:comment_id to make PUT or DELETE requests if _method parameter exists in query.

Parameter Description Type Default Required
message The comment's text string - yes
user_id The ID of the user to which the comment belongs integer - yes
application_id The applications'ID where the comment is posted integer - yes

If the create is successful, you get the following return.

Description Type Response Code
If the create succeeded Structure containing the id of the created comment 200

PUT

PUT requests can be made to /comments/:comment_id to update comment's data. If your client does not support the PUT method you can use POST requests with the parameter _method=PUT.

Parameter Description Type
message The comment's message string
user_id The ID of the user to which the commment belongs integer

If the update is successful, you get the following return.

Description Type Response Code
If update succeded boolean 200

DELETE

DELETE requests can be made to /comments/:comment_id to delete a comment or to /comments to delete all comments. If your client does not support the DELETE method you can use POST requests with the parameter _method=DELETE.

Parameter Description Type
before The maximum created date date (yyyy-MM-dd)
since The maximum created date date (yyyy-MM-dd)
user_id The user's ID to which the comment belongs integer
application_id The application's ID where the comment is posted integer

If the delete is successful, you get the following return.

Description Type Response Code
If delete succeded boolean 200
